虚幻引擎基础概念

发表于 2024-05-09 分类于 游戏引擎 本文字数： 1.7k 阅读时长 ≈ 6 分钟
虚幻引擎核心概念入门：蓝图、Actor、Pawn、GameMode、PlayerController等

记录学习虚幻引擎过程中的一些核心概念，主要是为了方便自己回顾。从蓝图开始，逐步深入到C++编程。

虚幻引擎概述

虚幻引擎（Unreal Engine，简称UE）是Epic Games开发的游戏引擎，目前已发展到UE5。主要特点： - 强大的渲染能力（Nanite虚拟几何体、Lumen全局光照） - 蓝图可视化编程 - 跨平台支持（PC、主机、移动设备） - 开源（C++源码可获取）

核心概念

1. Actor

Actor是虚幻引擎中最基本的对象，可以放置在关卡中。特点： - 有位置、旋转、缩放（Transform） - 可以被蓝图或C++创建和销毁 - 可以附加组件（Component）

Actor的生命周期： 1. 构造函数（Constructor） 2. PostInitializeComponents() 3. BeginPlay() 4. Tick()（每帧调用） 5. EndPlay() 6. 析构函数

2. Pawn

Pawn是Actor的子类，可以被Controller控制。用于表示玩家或AI控制的角色。

Pawn vs Actor： - Pawn可以被Controller控制 - Pawn可以接收输入 - Pawn有MovementComponent用于移动

3. Character

Character是Pawn的子类，专门用于表示人形角色。内置： - CharacterMovementComponent - CapsuleComponent（碰撞） - SkeletalMeshComponent（骨骼网格）

常用功能： - 行走、奔跑、跳跃 - 蹲伏、游泳、飞行 - 网络同步

4. Controller

Controller负责控制Pawn的行为。分为两种：

PlayerController：玩家控制

AIController：AI控制

5. GameMode

GameMode定义游戏规则，包括： - 玩家数量和类型 - 游戏开始、结束条件 - 胜负判定

重要函数： - InitGame()：游戏初始化 - BeginPlay()：游戏开始 - Tick()：每帧更新 - EndPlay()：游戏结束

6. GameState

GameState存储游戏状态信息，会同步给所有客户端（多人游戏）。

包含信息： - 游戏进行时间 - 当前回合/关卡 - 团队分数 - 玩家列表

7. PlayerState

PlayerState存储玩家状态，每个玩家一个。

包含信息： - 玩家名称 - 分数、击杀数、死亡数 - Ping值 - 当前Pawn

8. Level

Level（关卡）是游戏世界的容器，包含： - 地形、静态网格 - Actor、Pawn - 光照、雾 - 体积（Volume）

9. World

World是最高级别的容器，包含一个或多个Level。

World类型： - Game World：游戏世界 - Editor World：编辑器世界 - PIE World（Play In Editor）：编辑器中运行的游戏世界

蓝图系统

蓝图是虚幻引擎的可视化编程系统。

蓝图类型

  1. Blueprint Class：继承自C++类的蓝图
  2. Blueprint Function Library：函数库
  3. Blueprint Interface：接口
  4. Blueprint Macro Library：宏库

常用节点

事件节点： - Event BeginPlay - Event Tick - Event ActorBeginOverlap - Event KeyPressed

执行流节点： - Branch（if-else） - ForLoop - Sequence（顺序执行） - Delay

变量节点： - Get/Set - Cast To（类型转换）

函数节点： - PrintString - SpawnActor - DestroyActor

组件（Component）

组件可以附加到Actor上，提供特定功能。

常见组件

Scene Component： - 有Transform - 可以附加到其他Scene Component - 如：StaticMeshComponent、CameraComponent

Actor Component： - 没有Transform - 提供逻辑功能 - 如：InputComponent、ActorComponent

网络模式

Standalone：单人游戏

Listen Server：主机同时是服务器和客户端

Dedicated Server：专用服务器

同步方式

Replicate：在蓝图中设置Replicate

RPC（Remote Procedure Call）： - Server RPC（客户端调用，服务器执行） - Client RPC（服务器调用，客户端执行） - Multicast RPC（服务器调用，所有客户端执行）

总结

虚幻引擎的核心概念构成了游戏开发的基础： - Actor：世界中的对象 - Pawn：可控制的角色 - Controller：控制逻辑 - GameMode：游戏规则 - Component：功能模块 - Blueprint：可视化编程

学习虚幻引擎的过程中，最大的体会是：要理解框架的设计思想，而不是死记硬背API。蓝图很适合快速原型，但复杂逻辑还是用C++更清晰。
