发表于 2024-01-15 分类于 工具 本文字数： 2.4k 阅读时长 ≈ 9 分钟
Git常用命令与使用场景记录

主要记录一些常用的Git命令用法，这个笔记参考了黑马程序员的教程，并结合自己的实际使用经验

基础配置

git config - 配置用户信息

使用场景：第一次使用Git时需要配置，或者在不同项目中使用不同身份信息时。

基础操作

git init - 初始化仓库

使用场景：开始一个新项目时，或者想把一个已有项目纳入版本控制。

git add - 添加到暂存区

使用场景：修改代码后，准备提交前，需要将修改添加到暂存区。我通常使用git add .一次性添加所有修改。

git commit - 提交更改

使用场景：完成一个功能或修复一个bug后提交。如果提交信息写错了，可以用--amend修改。

git status - 查看状态

使用场景：随时查看哪些文件被修改、哪些文件在暂存区，避免误提交。

git log - 查看提交历史

使用场景：查看项目历史、追踪bug来源、找某个功能的提交记录。

分支管理

git branch - 分支操作

使用场景：开发新功能时创建分支，保持主分支稳定。

git checkout / git switch - 切换分支

使用场景：在不同分支间切换开发。我常用git checkout -b快速创建并切换分支。

远程操作

git remote - 管理远程仓库

使用场景：关联本地仓库和GitHub/GitLab远程仓库。

git clone - 克隆远程仓库

使用场景：第一次获取远程项目代码时使用。

git pull / git push - 拉取和推送

使用场景：团队协作时，开发完功能后push到远程，开始前pull最新代码。我通常在push前会先pull --rebase，保持提交历史整洁。

合并与冲突解决

git merge - 合并分支

使用场景：完成功能开发后，将特性分支合并回主分支。--no-ff可以保留分支历史。

git rebase - 变基

使用场景：想要保持线性提交历史时。我常用git rebase -i来整理提交，把多个小提交合并成一个。

撤销与回退

git reset - 重置到指定状态

使用场景：提交出错需要撤销时使用。--hard会丢失修改，使用时需要谨慎。

git revert - 撤销指定提交

使用场景：已经push到远程的提交需要撤销时使用。与reset不同，revert会创建新的提交来撤销更改，不会改变历史。

暂存与标签

git stash - 暂存修改

使用场景：正在开发功能时，需要切换到其他分支修复bug。我常用这个命令来临时保存工作进度。

git tag - 标签管理

使用场景：发布版本时打标签，方便后续回溯。
